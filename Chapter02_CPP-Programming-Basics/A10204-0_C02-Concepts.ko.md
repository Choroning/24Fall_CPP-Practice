## 1. 첫 번째 C++ 프로그램

### 1.1 프로그램 구조

모든 C++ 프로그램은 기본적인 구조를 따른다. 가장 간단한 완전한 프로그램을 한 줄씩 주석으로 설명하면 다음과 같다:

```cpp
// hello.cpp
#include <iostream>       // C++ 입출력 스트림 라이브러리
using namespace std;      // std 네임스페이스 사용

int main()                // 진입점 함수
{
    cout << "Hello" << endl;   // 콘솔에 출력
    return 0;             // OS에 종료 코드 반환
}
```

```
C++ 프로그램 구조
├── 전처리기 지시문          #include <iostream>
├── 네임스페이스 선언        using namespace std;
├── main 함수               int main() { ... }
│   ├── 실행문              cout << "Hello" << endl;
│   └── 반환문              return 0;
└── 기타 함수               (이후 장에서 학습)
```

| 구성 요소 | 역할 |
|:---------|:-----|
| `#include <iostream>` | 입출력 스트림 라이브러리를 포함하여 `cout`과 `cin`을 사용할 수 있게 함 |
| `using namespace std;` | `std::` 접두사 없이 `std` 이름을 직접 사용할 수 있게 함 |
| `int main()` | 진입점 정의; OS가 가장 먼저 호출하는 함수 |
| `{ ... }` | 중괄호로 함수 본문을 구분 |
| `cout << "Hello" << endl;` | `"Hello"`를 출력한 뒤 줄바꿈을 콘솔에 출력 |
| `return 0;` | 정상 종료를 의미하는 종료 코드 0을 OS에 반환 |

> **핵심:** 함수 내부의 모든 문장은 세미콜론(`;`)으로 끝나야 한다. 전처리기 지시문(`#include`)에는 세미콜론을 붙이지 **않는다**.

### 1.2 주석

주석은 컴파일러가 무시하며, 사람이 읽기 위해 존재한다. C++은 C와 동일한 주석 형식을 사용한다.

| 종류 | 문법 | 범위 |
|:----|:----|:----|
| 블록 주석 | `/* ... */` | 여러 줄에 걸칠 수 있음 |
| 줄 주석 | `// ...` | `//`부터 줄 끝까지 |

```cpp
/* 이것은 블록 주석이다.
   여러 줄에 걸칠 수 있다. */

// 이것은 줄 주석이다.

int x = 10;  // 코드 뒤에 붙이는 인라인 주석
```

> **경고:** 블록 주석은 **중첩할 수 없다**. `/* 외부 /* 내부 */ 아직 안쪽? */`는 컴파일 에러를 일으킨다. `*/`가 첫 번째 매칭에서 주석을 닫기 때문이다.

### 1.3 `main` 함수

`main` 함수는 모든 C++ 프로그램의 **진입점(entry point)** 이다. OS가 컴파일된 프로그램을 실행하면, `main`부터 실행이 시작된다.

| 형태 | 표준? | 비고 |
|:----|:----:|:----|
| `int main()` | 예 | 표준 형태; `int`를 반환 |
| `int main(void)` | 예 | 명시적 빈 매개변수 목록 |
| `void main()` | **아니오** | 비표준; 일부 컴파일러가 허용하나 이식 불가 |

```cpp
int main()        // 표준 형태
{
    return 0;     // 0 = 성공
}
```

C++에서 `main` 끝에 `return 0;`을 생략하면 컴파일러가 자동으로 `0`을 반환한다. 그러나 명확성을 위해 명시적으로 작성하는 것이 좋다.

> **경고:** `void main()`은 표준 C++이 **아니다**. 일부 컴파일러(예: 구형 MSVC)가 허용하지만, 이식성을 위해 피해야 한다. 항상 `int main()`을 사용한다.

## 2. 콘솔 출력

### 2.1 `cout`과 `<<`

`cout`은 표준 출력(화면)에 연결된 `ostream` 타입의 미리 정의된 객체이다. `<<` 연산자는 **스트림 삽입 연산자** — 데이터를 출력 스트림에 삽입한다.

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Hello, C++!" << endl;
    return 0;
}
```

| 구성 요소 | 설명 |
|:---------|:----|
| `cout` | 화면에 연결된 `ostream` 객체 (`<iostream>`에 선언됨) |
| `<<` | 스트림 삽입 연산자 — 데이터를 출력 스트림에 전달 |
| `endl` | 줄바꿈 문자를 삽입하고 출력 버퍼를 **비움(flush)** |

**체이닝** — 하나의 문장에서 여러 `<<` 연산자를 연결할 수 있다:

```cpp
int age = 20;
double gpa = 3.8;

cout << "나이: " << age << ", 학점: " << gpa << endl;
// 출력: 나이: 20, 학점: 3.8
```

C의 `printf`와 달리, `cout`은 출력할 데이터의 **타입을 자동으로 인식** 한다. `%d`나 `%f` 같은 형식 지정자가 필요 없다.

```cpp
// C 스타일 — 형식을 지정해야 함
printf("나이: %d, 학점: %.1f\n", age, gpa);

// C++ 스타일 — 타입이 자동 인식됨
cout << "나이: " << age << ", 학점: " << gpa << endl;
```

> **팁:** `cout`은 타입 안전하다. 잘못된 타입을 전달하면 컴파일러가 안전하게 변환하거나 컴파일 오류를 발생시킨다. `printf`의 경우 형식 지정자가 불일치하면 정의되지 않은 동작이 발생한다.

#### 현대적 출력 대안: `std::format`과 `std::print`

> **[C++20]** 이 기능은 C++20 이상이 필요합니다.

C++20에서는 Python 스타일의 문자열 포맷팅을 제공하는 `std::format`이 도입되었다. `cout`의 타입 안전성과 서식 문자열의 가독성을 결합한다:

```cpp
#include <format>
#include <iostream>
using namespace std;

int age = 20;
double gpa = 3.8;

// C++20: std::format은 서식이 적용된 문자열을 반환
cout << format("나이: {}, 학점: {:.1f}", age, gpa) << endl;
// 출력: 나이: 20, 학점: 3.8
```

> **[C++23]** 이 기능은 C++23 이상이 필요합니다.

C++23에서는 `cout` 없이 직접 출력하는 `std::print`가 추가되었다:

```cpp
#include <print>

int age = 20;
double gpa = 3.8;

// C++23: std::print — 서식 구문으로 직접 출력
std::print("나이: {}, 학점: {:.1f}\n", age, gpa);
// 출력: 나이: 20, 학점: 3.8
```

| 방식 | 표준 | 헤더 | 타입 안전 | 서식 제어 |
|:----|:----|:----|:--------:|:--------:|
| `printf` | C | `<cstdio>` | 아니오 | `%d`, `%f` |
| `cout <<` | C++98 | `<iostream>` | 예 | 조작자(manipulator) |
| `std::format` | C++20 | `<format>` | 예 | `{}` 자리 표시자 |
| `std::print` | C++23 | `<print>` | 예 | `{}` 자리 표시자 |

### 2.2 `endl` vs `'\n'`

`endl`과 `'\n'` 모두 줄바꿈을 생성하지만, 중요한 차이가 하나 있다:

| 방법 | 줄바꿈? | 버퍼 비움? |
|:----|:------:|:--------:|
| `endl` | 예 | 예 |
| `'\n'` | 예 | 아니오 |

```cpp
cout << "Line 1" << endl;   // 줄바꿈 + 버퍼 비움
cout << "Line 2" << '\n';   // 줄바꿈만 (버퍼 비우지 않음)
cout << "Line 3\n";         // 위와 동일 (문자열 리터럴 안의 줄바꿈)
```

**버퍼 비움(flush)** 이란 출력 버퍼에 있는 데이터를 즉시 화면에 쓰는 것이다. 비우지 않으면 데이터가 버퍼가 가득 차거나 프로그램이 종료될 때까지 버퍼에 남아 있을 수 있다.

> **팁:** 성능이 중요한 코드(예: 수백만 줄 출력)에서는 `endl`보다 `'\n'`을 선호한다. `endl`이 반복적으로 버퍼를 비우면 출력 속도가 크게 느려질 수 있다.

### 2.3 `<<` 연산자 세부사항

`<<` 연산자는 모든 기본 타입을 출력할 수 있다:

| 타입 | 예시 |
|:----|:----|
| `bool` | `cout << true;` → `1` (`boolalpha` 사용 시 `true`) |
| `char` | `cout << 'A';` → `A` |
| `int` | `cout << 42;` → `42` |
| `double` | `cout << 3.14;` → `3.14` |
| `string` | `cout << "Hello";` → `Hello` |

**식과 함수 호출** 을 `<<`와 함께 직접 사용할 수 있다:

```cpp
int a = 5, b = 3;
cout << a + b << endl;         // 출력: 8
cout << (a > b) << endl;       // 출력: 1 (true)
```

**연산자 우선순위 문제** — `<<` 연산자는 비교 연산자(`>`, `<`, `>=`, `<=`)보다 **우선순위가 낮다**:

```cpp
cout << 5 > 3;            // 오류 — (cout << 5) > 3 으로 해석됨
cout << (5 > 3) << endl;  // 정상 — 출력: 1
```

`cout << 5 > 3`은 `(cout << 5) > 3`으로 해석되어, `ostream` 객체를 `3`과 비교하려 하므로 타입 오류가 발생한다.

> **경고:** 비교 연산이나 산술 연산의 결과를 출력할 때는 항상 **괄호** 로 감싸서 우선순위 문제를 방지한다: `cout << (5 > 3)`.

## 3. 콘솔 입력

### 3.1 `cin`과 `>>`

`cin`은 표준 입력(키보드)에 연결된 `istream` 타입의 미리 정의된 객체이다. `>>` 연산자는 **스트림 추출 연산자** — 입력 스트림에서 데이터를 추출한다.

```cpp
#include <iostream>
using namespace std;

int main() {
    int num;
    cout << "숫자를 입력하세요: ";
    cin >> num;
    cout << "입력한 값: " << num << endl;
    return 0;
}
```

| 구성 요소 | 설명 |
|:---------|:----|
| `cin` | 키보드에 연결된 `istream` 객체 (`<iostream>`에 선언됨) |
| `>>` | 스트림 추출 연산자 — 입력 스트림에서 데이터를 읽음 |

**`>>`가 입력을 읽는 방법:**
- 선행 공백(스페이스, 탭, 줄바꿈)을 건너뛴다.
- 다음 공백 구분자까지 문자를 읽는다.
- 읽은 문자를 대상 변수의 타입으로 변환한다.

**체이닝** — 여러 `>>` 연산자를 연결할 수 있다:

```cpp
int a, b;
cout << "두 수를 입력하세요: ";
cin >> a >> b;    // 공백으로 구분된 두 정수를 읽음
cout << "합: " << a + b << endl;
```

```
입력:  10 20
출력: 합: 30
```

> **핵심:** `cin >> a >> b;`는 공백/탭/줄바꿈으로 구분된 두 개의 토큰을 입력에서 읽는다. 첫 번째 토큰은 `a`에, 두 번째는 `b`에 저장된다.

### 3.2 `getline`

`>>` 연산자는 다음 공백까지만 읽으므로, 공백이 포함된 문자열을 읽기에는 적합하지 않다. `getline` 함수는 공백을 포함한 **한 줄 전체** 를 읽는다.

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string name;
    cout << "이름을 입력하세요: ";
    getline(cin, name);
    cout << "안녕하세요, " << name << "!" << endl;
    return 0;
}
```

```
입력:  홍길동
출력: 안녕하세요, 홍길동!
```

**`>>`와 `getline` 혼용** — 흔한 함정:

`>>`가 값을 읽으면 뒤따르는 줄바꿈(`'\n'`)이 입력 버퍼에 남는다. 이후 `getline`을 호출하면 이 남은 줄바꿈을 읽고 빈 문자열을 반환한다.

**해결 방법:** `getline`을 호출하기 전에 `cin.ignore()`를 사용하여 남은 줄바꿈을 버린다.

```cpp
int age;
string name;

cout << "나이를 입력하세요: ";
cin >> age;                    // 숫자를 읽고, '\n'은 버퍼에 남음

cin.ignore();                  // 남은 '\n'을 버림

cout << "이름을 입력하세요: ";
getline(cin, name);            // 이제 올바르게 읽힘
cout << age << " " << name << endl;
```

> **경고:** `>>`와 `getline` 사이에 `cin.ignore()`를 빠뜨리는 것은 C++ 초보자가 가장 흔히 하는 실수 중 하나이다. `getline` 호출이 남은 줄바꿈을 즉시 읽어 "건너뛴 것처럼" 보인다.

**문제 발생 과정을 단계별로 시각화:**

```
사용자 입력: 25[Enter]홍길동[Enter]

1단계: cin >> age;
  버퍼 (이전): "25\n홍길동\n"
  읽은 값: 25
  버퍼 (이후): "\n홍길동\n"    ← '\n'이 남아 있음!

2단계 (cin.ignore() 없이):  getline(cin, name);
  버퍼 (이전): "\n홍길동\n"
  읽은 값: ""                    ← 첫 번째 '\n'까지 읽음 (즉시 만남)
  버퍼 (이후): "홍길동\n"
  결과: name은 빈 문자열!

2단계 (cin.ignore() 사용):  getline(cin, name);
  cin.ignore()가 '\n'을 버림
  버퍼 (이전): "홍길동\n"
  읽은 값: "홍길동"              ← 정상!
  버퍼 (이후): ""
```

**더 안전한 버전:** 최대한 안전하게 하려면, 단일 문자가 아닌 다음 줄바꿈까지의 모든 내용을 버리는 `cin.ignore(numeric_limits<streamsize>::max(), '\n')`를 사용한다:

```cpp
#include <limits>   // numeric_limits에 필요

cin >> age;
cin.ignore(numeric_limits<streamsize>::max(), '\n');   // 나머지 줄 전체 버림
getline(cin, name);
```

### 3.3 `cin` vs `scanf`

| 특성 | `cin` (C++) | `scanf` (C) |
|:----|:-----------|:-----------|
| 타입 안전성 | 타입 자동 인식 | 형식을 지정해야 함 (`%d`, `%f`) |
| 형식 지정자 | 불필요 | 필수 |
| 주소 연산자 `&` | 기본 타입에 불필요 | 필수 (`scanf("%d", &x)`) |
| 공백 포함 문자열 | `getline` 사용 | `fgets` 사용 |
| 오류 처리 | 스트림 상태 확인 (`cin.fail()`) | 반환값 확인 |

```cpp
// C 스타일
int x;
scanf("%d", &x);      // %d와 &를 사용해야 함

// C++ 스타일
int x;
cin >> x;              // 형식 지정자 없음, & 없음
```

> **팁:** C++에서는 `scanf`/`printf`보다 `cin`/`cout`을 선호한다. 타입 안전하고, 사용자 정의 타입으로 확장 가능하며, C++ 스트림과 자연스럽게 통합된다.

## 4. 네임스페이스

### 4.1 개념

**네임스페이스(namespace)** 는 식별자(이름)에 범위를 제공하는 선언 영역이다. 주된 목적은 **이름 충돌을 방지** 하는 것이다 — 서로 다른 라이브러리가 동일한 이름의 함수나 클래스를 정의할 수 있으며, 네임스페이스가 이들을 분리한다.

C++ 표준 라이브러리는 모든 이름을 `std` 네임스페이스 안에 배치한다.

```cpp
#include <iostream>

int main() {
    std::cout << "Hello" << std::endl;   // 완전한 한정 이름
    return 0;
}
```

| 문법 | 의미 |
|:----|:----|
| `std::cout` | `std` 네임스페이스에 정의된 `cout` |
| `std::endl` | `std` 네임스페이스에 정의된 `endl` |
| `std::string` | `std` 네임스페이스에 정의된 `string` 클래스 |

`std::` 접두사가 없으면 컴파일러가 `cout`의 정의 위치를 알 수 없어 오류를 보고한다.

### 4.2 `using namespace std`

모든 표준 라이브러리 이름 앞에 `std::`를 쓰지 않으려면 **using 지시문** 을 사용한다:

```cpp
using namespace std;   // std의 모든 이름을 현재 범위로 가져옴
```

이 선언 이후에는 `cout`, `cin`, `endl`, `string`, `vector` 등을 직접 사용할 수 있다.

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "std:: 접두사가 필요 없음" << endl;
    return 0;
}
```

**선택적 using 선언** — 특정 이름만 가져오기:

```cpp
#include <iostream>
using std::cout;
using std::endl;

int main() {
    cout << "cout과 endl만 가져옴" << endl;
    // cin은 여전히 std::cin으로 써야 함
    return 0;
}
```

| 방법 | 장점 | 단점 |
|:----|:----|:----|
| `std::cout` (완전 한정) | 모호함 없음; 출처 명확 | 장황함 |
| `using namespace std;` | 간결; 편리 | 대규모 프로젝트에서 이름 충돌 가능 |
| `using std::cout;` (선택적) | 편의성과 안전성의 균형 | 각 이름을 개별적으로 나열해야 함 |

> **팁:** 작은 프로그램과 학습용 예제에서는 `using namespace std;`를 사용해도 무방하다. 대규모 프로젝트와 헤더 파일에서는 이름 충돌을 방지하기 위해 완전 한정 이름이나 선택적 `using` 선언을 선호한다.

## 5. C++ 문자열

C++은 C 스타일 문자 배열(`char[]`)의 현대적이고 안전한 대안으로 `std::string` 클래스를 제공한다.

```cpp
#include <string>     // std::string 사용에 필요
#include <iostream>
using namespace std;

int main() {
    string greeting = "Hello, C++!";
    cout << greeting << endl;
    return 0;
}
```

**`std::string` vs C 스타일 `char[]`:**

| 특성 | `std::string` (C++) | `char[]` (C) |
|:----|:-------------------|:-------------|
| 헤더 | `#include <string>` | `#include <cstring>` |
| 선언 | `string s = "Hello";` | `char s[] = "Hello";` |
| 연결 | `s1 + s2` | `strcat(s1, s2)` |
| 비교 | `s1 == s2` | `strcmp(s1, s2) == 0` |
| 길이 | `s.length()` 또는 `s.size()` | `strlen(s)` |
| 동적 크기 | 예 (자동으로 확장) | 아니오 (선언 시 고정) |
| 안전? | 예 (범위 관리됨) | 아니오 (수동 버퍼 관리) |

**주요 연산:**

```cpp
string a = "Hello";
string b = "World";

// 연결
string c = a + ", " + b + "!";   // "Hello, World!"

// 비교
if (a == b) { /* ... */ }         // == 로 자연스럽게 비교

// 길이
cout << c.length() << endl;       // 13

// getline으로 입력 (공백 포함 한 줄 전체 읽기)
string line;
getline(cin, line);

// 개별 문자 접근
cout << a[0] << endl;             // 'H'
```

> **핵심:** `std::string`은 여기서 간략히 소개한다. 자세한 문자열 연산, 메서드, 반복자 등은 이후 장에서 다룬다. 지금은 C++에서 `string`이 `char[]`보다 안전하고 편리하다는 점만 알아두면 된다.

## 6. `auto`를 사용한 현대적 변수 선언

C++11부터 `auto` 키워드를 사용하면 컴파일러가 초기화 값으로부터 변수의 **타입을 추론** 한다. 이는 중복을 줄이고 코드 유지보수성을 높인다.

```cpp
auto x = 42;           // int
auto pi = 3.14;        // double
auto name = "Hello"s;  // std::string (s 접미사, "using namespace std::literals" 필요)
auto greeting = string("Hello");  // std::string

// 복잡한 타입에서 특히 유용
vector<int> nums = {1, 2, 3};
auto it = nums.begin();   // vector<int>::iterator — auto로 훨씬 간결
```

**`auto` 사용 시점:**

| `auto` 사용 | 명시적 타입 사용 |
|:-----------|:--------------|
| 우변에서 타입이 명확할 때 | 타입이 불명확하여 가독성이 떨어질 때 |
| 복잡한 타입 (반복자, 람다 등) | 의도적으로 다른 타입을 원할 때 (예: `int x = 3.14;`로 절삭) |
| 범위 기반 for 루프: `for (auto& item : vec)` | 공개 API / 함수 시그니처 (명확성이 중요) |

> **참고:** `auto`는 변수가 타입이 없거나 동적 타입이라는 뜻이 아니다. 타입은 **컴파일 시점** 에 결정되며 이후 고정된다. C++은 여전히 정적 타입 언어이다.

## 요약

| 개념 | 핵심 정리 |
|:----|:---------|
| 프로그램 구조 | `#include <iostream>` → `using namespace std;` → `int main()` → 실행문 → `return 0;` |
| 주석 | `/* 블록 */` 과 `// 줄`; C와 동일; 블록 주석은 중첩 불가 |
| `main` 함수 | 진입점; `int` 반환; `void main()`은 비표준 |
| `cout`과 `<<` | 출력용 `ostream` 객체; 스트림 삽입 연산자; 타입 자동 인식; 체이닝 가능 |
| `endl` vs `'\n'` | `endl`은 버퍼를 비움; `'\n'`은 비우지 않음; 성능을 위해 `'\n'` 선호 |
| `<<` 우선순위 | 비교 연산자보다 우선순위 낮음; 괄호 사용: `cout << (5 > 3)` |
| `cin`과 `>>` | 입력용 `istream` 객체; 스트림 추출 연산자; 공백 건너뜀; 체이닝 가능 |
| `getline` | 공백 포함 한 줄 전체 읽기; `>>` 후 `getline` 전에 `cin.ignore()` 사용 |
| `cin` vs `scanf` | `cin`은 타입 안전, 형식 지정자 불필요, 기본 타입에 `&` 불필요 |
| 네임스페이스 | 이름 충돌 방지; `std`에 표준 라이브러리 포함; `std::cout` |
| `using namespace std` | `std::` 접두사 생략 단축어; `using std::cout`으로 선택적 가져오기 |
| `std::string` | `char[]`의 현대적 대안; `+`로 연결, `==`로 비교, `.length()`로 길이 확인 |
| `std::format` / `std::print` | 현대적 출력 서식 (C++20/C++23); `{}` 자리 표시자; `cout`처럼 타입 안전하면서 서식 문자열 사용 |
| `auto` | 컴파일러가 초기화 값에서 타입 추론 (C++11); 반복자와 복잡한 타입에 유용; 여전히 정적 타입 |
